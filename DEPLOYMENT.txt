CrowdAid - Deployment Guide (Making It Live)
==============================================

This guide helps you deploy CrowdAid on the internet for free.
Make sure the app works locally first (see SETUP.txt).

We use:
- Supabase  -> Database (free)
- Render    -> Backend hosting (free)
- Vercel    -> Frontend hosting (free)

Total cost: $0


STEP 1: Push Code to GitHub
------------------------------

1. Create a new repository on GitHub (github.com -> New Repository)
   - Name it something like "crowdaid"
   - Keep it public or private, both work

2. Push your code:

   git init
   git add .
   git commit -m "crowdaid project"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/crowdaid.git
   git push -u origin main


STEP 2: Set Up Supabase Database
-----------------------------------

If you already set this up from SETUP.txt, skip to step 3.
Otherwise follow SETUP.txt Step 2 to create a Supabase project.

Keep your DATABASE_URI ready (you will need it in the next step):
  postgresql://postgres.xxxxx:PASSWORD@host:5432/postgres


STEP 3: Generate a JWT Secret
--------------------------------

You need a strong random string (32+ characters) for JWT token signing.

Easy ways to generate one:
- Go to https://generate-random.org/api-key-generator and copy a key
- Or run this in terminal: openssl rand -base64 48
- Or just type a long random string yourself (at least 32 characters)

Save this string somewhere, you will paste it in the next step.


STEP 4: Deploy Backend on Render
-----------------------------------

1. Go to https://render.com and sign up (use GitHub login)

2. Click "New +" at the top -> select "Blueprint"

3. Connect your GitHub repository when prompted

4. Render will detect the render.yaml file and show a setup page

5. Fill in the environment variables it asks for:
   - DATABASE_URI     = your Supabase connection URI from step 2
   - JWT_SECRET       = the random string from step 3
   - CORS_ALLOWED_ORIGINS = https://placeholder.vercel.app
     (we will update this later after deploying frontend)
   - ADMIN_EMAIL      = your email for admin login
   - ADMIN_PASSWORD   = a strong password, at least 12 characters
   - ADMIN_PHONE      = your phone with +91 (example: +919876543210)

6. Click "Apply" and wait for deployment (takes about 5-10 minutes)

7. Once deployed, note your backend URL. It will look like:
   https://crowdaid-backend.onrender.com

8. Quick test: open this URL in your browser:
   https://crowdaid-backend.onrender.com/api/health
   You should see a JSON response with success: true

IMPORTANT NOTE: Render free tier puts your server to sleep after 15 minutes
of no activity. The first request after sleep takes about 30 seconds to
respond. This is normal for free hosting.


STEP 5: Deploy Frontend on Vercel
------------------------------------

1. Go to https://vercel.com and sign up (use GitHub login)

2. Click "Add New Project"

3. Import your GitHub repository

4. Configure the project settings:
   - Root Directory    : frontend
   - Framework Preset  : Vite
   - Build Command     : npm run build
   - Output Directory  : dist

5. Before deploying, add this environment variable:
   Click "Environment Variables" and add:
   - Name:  VITE_API_BASE_URL
   - Value: https://crowdaid-backend.onrender.com/api
   (use YOUR actual Render backend URL from step 4, with /api at the end)

6. Click "Deploy" and wait (takes about 2-3 minutes)

7. After deployment, note your frontend URL:
   https://crowdaid-xxxx.vercel.app
   (Vercel gives you a URL automatically)


STEP 6: Update CORS on Render (IMPORTANT!)
--------------------------------------------

Right now your backend does not allow requests from your Vercel frontend.
You need to fix this:

1. Go back to Render Dashboard (render.com)
2. Click on your crowdaid-backend service
3. Go to "Environment" tab
4. Find CORS_ALLOWED_ORIGINS and change its value to:
   https://crowdaid-xxxx.vercel.app
   (use your ACTUAL Vercel URL from step 5, no trailing slash)

5. Click "Save Changes"
6. Render will automatically redeploy with the new setting


STEP 7: Test Everything Live
-------------------------------

1. Open your Vercel URL in the browser

2. Wait a moment for the backend to wake up (if its sleeping)

3. Register a new account:
   - Fill in all fields
   - Check volunteer if you want

4. Create an SOS request:
   - Allow location access when the browser asks
   - Select an emergency type

5. Test admin login:
   - Log out
   - Log in with the ADMIN_EMAIL and ADMIN_PASSWORD you set in step 4
   - Check the admin dashboard for system stats

6. Test volunteer flow:
   - Register a second account as a volunteer
   - See nearby SOS requests and try responding

If anything does not work:
- Check Render logs: Dashboard -> your service -> Logs tab
- Check browser console: Press F12 -> Console tab
- Make sure CORS_ALLOWED_ORIGINS matches your EXACT Vercel URL
- Make sure VITE_API_BASE_URL on Vercel ends with /api


TIPS FOR KEEPING IT FREE
--------------------------

- Keep Render on the Free plan (cold starts are expected and normal)
- Keep Supabase on the Free plan (you get 500 MB storage, thats plenty)
- Keep Vercel on the Hobby plan (free for personal projects)
- No paid services or APIs are needed - everything runs on free tiers
- If you want to reduce cold starts on Render, you can set up a free
  uptime monitor (like UptimeRobot) to ping your /api/health endpoint
  every 14 minutes


OPTIONAL: Custom Domain
--------------------------

If you have your own domain name:
- On Vercel: Go to Project Settings -> Domains -> Add your domain
- On Render: Go to Service Settings -> Custom Domains
- Update CORS_ALLOWED_ORIGINS on Render to include your custom domain
